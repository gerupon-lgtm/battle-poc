# バトル方式 検証 PoC（self-contained）

`index.html` のターン制バトルUIと `rhythm-battle-poc.html` のリズム機構を流用し、
「はしけん」クイズと組み合わせた **2 案のバトルフロー** を検証するための独立フォルダです。

- **既存システムには一切手を加えていません。** すべてこの `app/` 配下で完結します。
- そのまま GitHub の別リポジトリへ push して動作させられる構成です（リポジトリ内だけで完結）。

## フォルダ構成
```
app/
├── index.html          方式選択ランディング
├── tantan.html         たんたん案
├── toshi.html          トシ案(改)
├── css/
│   ├── rhythm-battle-poc.css   (元POCからコピー)
│   └── battle-verify.css       (検証レイヤー)
├── js/
│   ├── rhythm-battle-poc.js    (元POPをコピー＋ラウンド終了フックを追加)
│   ├── enemy-loader.js         enemies.csv 読込(そらドラゴン固定 enemy_008)
│   ├── quiz-engine.js          クイズ出題(ランダム/正解必須/選択肢シャッフル/絞り込み)
│   ├── battle-core.js          HP・ログ・クイズ描画・リズム実行の共通基盤
│   ├── battle-tantan.js        たんたん案フロー
│   └── battle-toshi.js         トシ案(改)フロー
└── data/
    ├── enemies.csv
    └── hashiken-question-set.json
```

## 起動方法（重要）
`fetch` で CSV / JSON を読むため、**ローカルHTTPサーバ経由**で開いてください
（`file://` 直開きでは読み込みに失敗します）。

```bash
cd app
python3 -m http.server 8000
# → http://localhost:8000/index.html を開く
```
GitHub Pages にそのまま置いても動作します（ルートを `app/` にする）。

## 2 案のフロー

### たんたん案（`tantan.html`）
1. リズムラウンド（既定曲）を演奏 → 獲得スコアで次のクイズの**選択肢数が決まる**（高スコアほど少=易しい）
2. クイズ出題：正解=敵にダメージ／不正解=自分にダメージ
3. 解説は**バトル後にまとめて表示**（勝利時）
4. どちらかの HP が 0 になるまで 1〜3 を繰り返す

### トシ案（改）（`toshi.html`）
ページ上部で**リズムの役割**を選べます（PoC ではトシ案トップで選択）。**既定は「防御専用」**。

クイズ正解時の敵へのダメージは、**敵の元最大HPの30%を基準**に、
**±の乱数（負数も含む）を加算**して増減させます（`battle-toshi.js` の `CONFIG`）。
- `DAMAGE_BASE_RATIO: 0.30`（基準＝元最大HP×30%。そらドラゴンHP75なら約22）
- `DAMAGE_RANDOM_RANGE: 8`（±8の一様乱数を加算。0で固定）
- `DAMAGE_MIN: 1`（下限。敵を回復させない）
おおよそ3〜4回の正解で撃破できる目安です。※「守備力半減」ロジックは廃止（敵HPが本当に0にならない整合性の問題があったため）。

**攻撃型（既定）**
1. クイズ窓（2択）：正解=敵に**直接ダメージ**（元ターン制ロジック）／不正解=ダメージなし
2. 防御リズム：クリア=敵にダメージ／時間切れ=自分にダメージ
3. 解説は**勝利後**にまとめて表示（学びの導線）
4. どちらかの HP が 0 になるまで繰り返す

**防御専用**
1. クイズ窓（2択）：正解=敵に**直接ダメージ**（元ターン制ロジック）／不正解=ダメージなし
2. リズムは**防御専用**：敵の反撃（被ダメージ）を**獲得スコアで軽減**（高得点ほど被害小、十分高ければ無傷）
3. 解説は**勝利後**にまとめて表示
4. どちらかの HP が 0 になるまで繰り返す

## 敵へのダメージ（両案共通）
クイズ正解時の敵ダメージは `BattleCore.rollDamage` に共通化（`battle-tantan.js` / `battle-toshi.js` の `CONFIG.DAMAGE`）。
- 通常: **HPに依存しない絶対ダメージ基準** `attackBase`（既定22）に **±乱数（負数含む, `randomRange`）** を加算。
  → 敵HPが多いほど撃破に必要なターンが増える設計。目安: HP20≈1ターン(約64%が一撃)、HP33≈2、HP50≈3、HP75≈4。
- **かいしんのいちげき**: `critChance`（既定2%, 極低確率・将来パラメータ化）で発動し、
  **敵の元最大HPの70〜90%**（`critMinRatio`〜`critMaxRatio`）の大ダメージ。
- `min`: ダメージ下限（敵を回復させない）。
- 最弱キャラをほぼ確実に一撃にしたい場合は `attackBase` を 25〜28 程度に上げると一撃率が上がります。

## リズムの曲・タップパターン
- **曲**: 各ラウンド開始時にランダム選択（`battle-core.js` の `pickRandomSong`）。
- **タップパターン**: 順送り進行（`createPatternSequencer`）。
  - 順番: 基本 → 裏拍 → 技巧 → 余白 → ブレイク技巧
  - **クリアで次のパターンへ進む。時間切れ（未クリア）は同じパターンで再戦。**
  - ブレイク技巧をクリアした後はランダム。
- 選ばれた曲名・パターン名はプロンプトに表示されます。

## Android の振動（ハプティクス）
Android 端末では振動モードを既定で OFF にします（`battle-core.js`、UA に "Android" を含む場合に
ハプティクスのトグルを OFF＋無効化）。iOS/PC は元々振動非対応のため影響ありません。

## モバイル: リズム画面を一画面に
- スマホ幅(～560px)では、リズム中だけ画面全体を 100dvh のレイアウトに切り替え、
  **ヘッダー(ドラゴン)＋レーン＋こうげき＋スコア＋［次へ］が一画面に収まる**ようにしています。
- レーン(プレイ領域)は残り高さを自動で吸収し、端末に合わせて縦サイズが伸縮します（最小130px）。
- 省スペースのため、リズム中はタイトル/戻るリンク/下部コメント欄/流用元の戦闘ログ・敵ゲージ・曲選択を一時的に隠します。
- 調整したい場合の主な箇所(`css/battle-verify.css` 末尾): レーン最小高 `min-height: 130px`、
  隠す要素の指定(`.enemy-area` 等)。

## リズム結果の表示タイミング
- リズムラウンドの結果（撃破！／時間切れ・スコア・コンボ）を**表示したまま保持**し、
  ［次へ］を押してから次のターン（クイズ等）へ進みます。
- 二回目以降のリズムラウンド開始時には、**前ターンの結果表示を自動でクリア**します。

## 仕様・制約への対応
- 出題はランダム（`quiz-engine.js`）
- 選択肢を絞る場合も**正解は必須**、他はランダム抽出（`next(choiceCount)`）
- 選択肢の順番もシャッフル
- 敵キャラは `enemies.csv` から**ランダム出現**（26体。hp/attack/defense を使用、画像も各敵）
- リズムは**ラウンドごとに曲・タップパターンをランダム選択**。プレイ画面・スコア表示・コメント欄（成功/失敗/ダメージ等）を流用
- 現状のターン制バトルのフロントUIイメージ（敵スプライト＋HPバー＋自HPバー＋コメント欄）を残して流用

## 調整パラメータ
各フローの先頭 `CONFIG` で調整できます。
- `battle-tantan.js`：`SCORE_FOR_2_CHOICES` / `SCORE_FOR_3_CHOICES`（選択肢が絞られるスコア境界）、`CORRECT_BASE_DAMAGE`、`CLEAR_BONUS_DAMAGE` など
- `battle-toshi.js`：`QUIZ_CHOICES`（2 or 3）、`PLAYER_ATTACK`、守備半減、`TIMEOUT_*` など

## リズムエンジンへの変更点（最小・追加のみ）
`rhythm-battle-poc.js` のコピーに対し、ラウンド終了時に結果を通知するフックを 2 箇所追加しただけです。
- 撃破（クリア）時：`window.RhythmBridge.onRoundEnd({score, combo, cleared:true})`
- 時間切れ時：`window.RhythmBridge.onRoundEnd({score, combo, cleared:false})`

元の挙動（演奏・採点・表示）はそのまま。`window.RhythmBridge` が無ければ何もしません。

## 将来 DB 化の想定
クイズは本検証では JSON を読み込みますが、将来的には DB 格納予定。
